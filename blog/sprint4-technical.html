Problems and ways to solve them

A time I got blocked on a simple problem was...

During this course I was doing a test exercise and I didnt read the question properly. I couldnt solve it so I kept trying to google answers and I got really frustrated. I finally went back to the start and re-read it, then found it was really simple.

I guess the lesson was Less haste, More speed.

A time I solved a problem in an elegant way...

I wouldnt call my code elegant, but I have made an effort to tidy it as my understanding grows. The only time I can think of solving a problem well was breaking it up and getting it to run basic commands, and building from there.

Reflecctions

Looking at both of these situations, I would say the bigget lesson Ive learnt is to take the time to read the instructions thoroughly, and to keep things simple. I keep trying to go fast and making things worse and more complicated.

The main problem solving technique I feel confident using is breaking the problem up.

Comments on using the following techniques

Psuedocode

When your learning to code, it seems silly to write obvious statements on small pieces of code. The more I tie concepts together, the more I see the value in this. Its also handy when looking at other peoples code.

Trying something

I feel confident to just try things as im quite a kinetic learner.

Rubbger ducky method

I quite like talking to inanimate objects, so this is just an extension of something I already do.

Reading error messages

Not my strong point, but I have had some sucess with reading error messages.
